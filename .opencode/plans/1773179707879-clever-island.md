# Plan: Rewrite cbNP in Go

## Summary

Rewrite the macOS menu bar application `cbNP` from Python (651+141+128 lines, rumps/PyObjC/asyncio) to Go. The Go version will be simpler, fix all identified bugs, redesign the WebSocket protocol, and include a full CI/CD pipeline via GitHub Actions.

---

## Current Bugs & Issues Being Fixed

| # | Issue | Fix |
|---|-------|-----|
| 1 | Thread-unsafe global `current_track` (timer thread writes, asyncio thread reads) | Single goroutine owns `lastTrack` in the poll loop |
| 2 | `push_update` sends full payload every interval even when track unchanged | Only send on track change |
| 3 | Heartbeat missing `auth` field | `auth` in every message envelope |
| 4 | Changing endpoint in preferences doesn't reconnect WebSocket | Reconnect on config change |
| 5 | Artwork errors silently swallowed (`except Exception` with no logging) | Log all artwork errors |
| 6 | `Pillow` is unused dependency | Removed (Go has no equivalent need) |
| 7 | `pyinstaller` listed as runtime dependency | N/A in Go |
| 8 | No reconnect backoff (retries every 2s forever) | Exponential backoff: 1s -> 60s cap |
| 9 | File I/O for every log line (open/write/close) | `slog` with single writer |
| 10 | No log rotation | Rotate on size (5MB) |
| 11 | `cbNP.spec` references non-existent `cbNP-icon.icns` | Build script generates `.icns` properly |
| 12 | README schema says `track` but payload sends `name` | Redesigned protocol with `title` |
| 13 | No tests | Unit tests for config, protocol, transport, media parsing |

```go
// internal/transport/protocol.go - Redesigned protocol
type Envelope struct {
    Type    string          `json:"type"`              // "update" | "heartbeat"
    Auth    string          `json:"auth"`              // always present
    Version string          `json:"version"`           // "1"
    SentAt  time.Time       `json:"sent_at"`           // ISO 8601
    Payload json.RawMessage `json:"payload,omitempty"`
}

type UpdatePayload struct {
    Title   string `json:"title"`
    Artist  string `json:"artist"`
    Album   string `json:"album"`
    Artwork string `json:"artwork"`   // base64
    TrackID string `json:"track_id"`
    Source  string `json:"source"`    // "Music"|"Spotify"|"MediaRemote"
}
```

### Goroutine Architecture

```
main goroutine (macOS main thread)
  ‚îî‚îÄ‚îÄ systray.Run(onReady, onExit)
        ‚îî‚îÄ‚îÄ onReady: go app.Run(ctx)

app.Run goroutine (single select loop):
  ‚îú‚îÄ‚îÄ <-pollTicker.C     ‚Üí source.NowPlaying() ‚Üí if changed ‚Üí wsClient.Send(update)
  ‚îú‚îÄ‚îÄ <-heartbeatTicker.C ‚Üí wsClient.Send(heartbeat)
  ‚îú‚îÄ‚îÄ <-wsClient.State() ‚Üí update tray icon/tooltip
  ‚îî‚îÄ‚îÄ <-ctx.Done()       ‚Üí graceful shutdown

wsClient.Run goroutine:
  ‚îî‚îÄ‚îÄ dial ‚Üí read sendCh ‚Üí write to conn ‚Üí on error ‚Üí reconnect with backoff
```

Key: only 2 goroutines matter (app loop + ws client). No global state. No mutexes. Channel-based communication.

## Implementation Steps

### Phase 1: Project Scaffold
1. Create `go.mod`, `go.sum` (init module)
2. Create directory structure (`cmd/cbnp/`, `internal/*/`)
3. Add `.golangci.yml`
4. Add `Makefile` (build, test, lint, run, clean)
5. Copy `assets/logo.png` to `internal/tray/logo.png` for embedding

### Phase 2: Config Package (`internal/config/`)
6. Implement `Config` struct with YAML tags
7. Implement `DefaultConfig()`, `Load()`, `Save()`, `Validate()`
8. Implement `MergeFlags()` for CLI override
9. Write `config_test.go`

### Phase 3: Media Package (`internal/media/`)
10. Define `Track` type and `Source` interface
11. Implement `AppleScriptSource` (Music + Spotify via `osascript`)
12. Implement `MediaRemoteSource` (perl adapter shelling)
13. Implement `artwork.go` (hex/URL/raw normalization)
14. Write tests (mock exec.Command)

### Phase 4: Transport Package (`internal/transport/`)
15. Implement `protocol.go` (Envelope, UpdatePayload, HeartbeatPayload, marshal/unmarshal)
16. Implement `client.go` (WebSocket connect, reconnect with backoff, Send channel, State channel)
17. Write `protocol_test.go` (round-trip serialization)
18. Write `client_test.go` (mock WebSocket server via httptest)

### Phase 5: Tray Package (`internal/tray/`)
19. Implement `icon.go` with `go:embed`
20. Implement `tray.go` (systray setup, menu items, source submenu, event channels)

### Phase 6: App Orchestrator (`internal/app/`)
21. Implement `app.go` (Run loop with select, poll ticker, heartbeat ticker, state handling)
22. Wire Source + Client + Tray together
23. Write `app_test.go` (mock Source and Client)

### Phase 7: Main Entry Point (`cmd/cbnp/`)
24. Implement `main.go` (pflag parsing, config load, version flag, systray.Run)
25. Version injection via `-ldflags`

### Phase 8: Build System
26. Create `scripts/Info.plist.template`
27. Create `scripts/build-app.sh` (.app bundle creation + icns generation)
28. Update `Makefile`

### Phase 9: CI/CD
29. Create `.github/workflows/ci.yml` (lint, test, build matrix, release)

### Phase 10: Cleanup
30. Update `README.md` for the Go version
31. Remove old Python files (`cbNP.py`, `helper.py`, `mediaremote.py`, `cbNP.spec`, `pyproject.toml`, `uv.lock`)
32. Update `.gitignore` for Go
